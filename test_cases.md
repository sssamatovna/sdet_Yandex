
# ТЕСТ-ПЛАН ДЛЯ YANDEX DISK API

---
## ПОЗИТИВНЫЕ СЦЕНАРИИ (POSITIVE CASES) 

ТС-01: Создание новой папки
Приоритет: Critical
Тип: Позитивный
Предусловия:
  1. Пользователь авторизован (есть валидный OAuth токен).
  2. Токен передаётся в заголовке: "Authorization: OAuth <token>".
  3. Папка с указанным именем {name} не существует на диске.
Шаги:
  1. Отправить PUT запрос на адрес: https://cloud-api.yandex.net/v1/disk/resources
  2. Передать Query-параметр: path={name}
Ожидаемый результат:
  1. Статус-код ответа: 201 Created.
  2. JSON-ответ не пустой, содержит поля "href", "method" (GET).
  3. Папка физически создана на Диске.


ТС-02: Удаление папки
Приоритет: Critical
Тип: Позитивный
Предусловия:
  1. Пользователь авторизован (есть валидный OAuth токен).
  2. Токен передаётся в заголовке: "Authorization: OAuth <token>".
  3. Существует папка с именем {name} (создать её перед тестом).
Шаги:
  1. Отправить DELETE запрос на адрес: https://cloud-api.yandex.net/v1/disk/resources
  2. Передать Query-параметр: path={name}
Ожидаемый результат:
  1. Статус-код ответа: 204 No Content (или 202 Accepted, если операция ставится в очередь).
  2. Папка успешно перемещена в Корзину и пропала из основного списка ресурсов.


ТС-03: Восстановление папки из корзины
Приоритет: Critical
Тип: Позитивный
Предусловия:
  1. Пользователь авторизован (есть валидный OAuth токен).
  2. Токен передаётся в заголовке: "Authorization: OAuth <token>".
  3. Создать папку {name}.
  4. Удалить папку {name} (переместив в корзину).
  5. Отправить GET запрос на адрес v1/disk/trash/resources/?path=app:/{name}
  6. Сохранить значение resource_id.
Шаги:
  1. Отправить PUT запрос на адрес: https://cloud-api.yandex.net/v1/disk/trash/resources/restore
  2. Передать Query-параметр: path={name}
Ожидаемый результат:
  1. Статус-код ответа: 201 Created.
  2. JSON-ответ содержит валидную структуру с полями "href" и "method".
  3. Папка успешно восстановлена и снова доступна в основном корневом каталоге.



## НЕГАТИВНЫЕ ТЕСТЫ


ТС-04: Создание папки с уже существующим именем
Приоритет: High
Тип: Негативный
Предусловия:
  1. Пользователь авторизован (есть валидный OAuth токен).
  2. Токен передаётся в заголовке: "Authorization: OAuth <token>".
  3. Папка с указанным именем {name} уже существует на диске.
Шаги:
  1. Отправить PUT запрос на адрес: https://cloud-api.yandex.net/v1/disk/resources
  2. Передать Query-параметр: path={name}
Ожидаемый результат:
  1. Статус-код ответа: 409 Conflict.
  2. JSON-ответ содержит структуру ошибки:
     * error: "ResourceAlreadyExistsError"
     * description: "Платформа выдает ошибку уникальности пути"


ТС-05: Удаление несуществующей папки
Приоритет: Medium
Тип: Негативный
Предусловия:
  1. Пользователь авторизован (есть валидный OAuth токен).
  2. Токен передаётся в заголовке: "Authorization: OAuth <token>".
  3. Убедиться, что папки с именем {name} не существует на Диске.
Шаги:
  1. Отправить DELETE запрос на адрес: https://cloud-api.yandex.net/v1/disk/resources
  2. Передать Query-параметр: path={name}
Ожидаемый результат:
  1. Статус-код ответа: 404 Not Found.
  2. JSON-ответ содержит структуру ошибки:
     * error: "DiskNotFoundError" или "NotFoundError"


ТС-06: Восстановление несуществующего ресурса из корзины
Приоритет: Medium
Тип: Негативный
Предусловия:
  1. Пользователь авторизован (есть валидный OAuth токен).
  2. Токен передаётся в заголовке: "Authorization: OAuth <token>".
  3. В корзине нет папки {name}.
Шаги:
  1. Отправить PUT запрос на адрес: https://cloud-api.yandex.net/v1/disk/trash/resources/restore
  2. Передать Query-параметр: path={name}
Ожидаемый результат:
  1. Статус-код ответа: 404 Not Found.
  2. JSON-ответ содержит структуру ошибки:
     * error: "TrashNotFoundError" или "NotFoundError"
